# Analisis De Competencia - Cotiza

Consulta inicial: 2026-05-28.

Este documento no busca copiar productos. Sirve para entender el mercado, detectar ideas utiles y proteger el foco de Cotiza.

## Lectura rapida

Si, existen apps y programas cercanos a Cotiza.

La competencia se divide en tres grupos:

- Software profesional de obra, mediciones y presupuestos.
- Apps rapidas para presupuestos de reformas, oficios y contratistas.
- Herramientas generales de presupuestos, facturacion o gestion para autonomos.

La oportunidad de Cotiza esta en ser mas simple que un software tecnico de obra, pero mas controlado y confiable que una app que genera presupuestos con IA sin una base propia del usuario.

## Aprendizajes para hacer mas facil la carga

Este es el punto mas importante para Cotiza ahora: no mirar la competencia para competir, sino para entender como reducir friccion al cargar datos.

### 1. Cargar por oficio, no por modulo tecnico

En vez de obligar al usuario a pensar primero en "precios", "rendimientos" y "trabajos tipo", Cotiza puede ofrecer caminos guiados:

- Quiero cargar pintura.
- Quiero cargar electricidad.
- Quiero cargar fontaneria.
- Quiero cargar suelo o alicatado.
- Quiero cargar un trabajo propio.

Dentro de cada camino, Cotiza puede pedir solo lo necesario.

Ejemplo para pintura:

- Precio de pintura por litro.
- Rendimiento: cuantos m2 cubre 1 litro por mano.
- Precio hora de mano de obra.
- Horas estimadas por m2.
- Material auxiliar por m2.
- Margen habitual.

Resultado: Cotiza crea automaticamente los precios, rendimientos y un trabajo tipo inicial.

### 2. Usar asistentes cortos de carga

La competencia suele vender rapidez porque evita pantallas tecnicas largas. Cotiza puede aplicar eso sin perder control.

Asistente recomendado:

1. Elegir oficio.
2. Elegir trabajo frecuente.
3. Completar 5 a 8 datos simples.
4. Ver una vista previa del calculo.
5. Guardar como trabajo tipo editable.

Esto mantiene la base controlada, pero evita que el usuario tenga que entender toda la estructura desde el primer minuto.

### 3. Mostrar siempre la formula humana

La calculadora debe explicar el resultado con frases simples.

Ejemplo:

```text
Pintura necesaria:
25 m2 x 0,125 litros por m2 = 3,13 litros

Mano de obra:
25 m2 x 0,18 horas por m2 = 4,5 horas

Precio cliente:
Coste + margen + impuesto = total
```

Esto ayuda mas que esconder el calculo detras de una tabla.

### 4. Permitir empezar con ejemplos y luego reemplazar

Para una demo y para onboarding real, conviene que el usuario no empiece desde cero.

Cotiza deberia ofrecer:

- "Usar ejemplos de pintura".
- "Usar ejemplos de electricidad".
- "Usar ejemplos de fontaneria".
- "Despues cambio los precios por los mios".

La carga inicial se vuelve menos pesada porque el usuario corrige datos existentes en vez de crear todo desde una hoja vacia.

### 5. Agrupar precios relacionados

En lugar de una lista plana enorme, Cotiza puede mostrar grupos:

- Materiales.
- Mano de obra.
- Transporte.
- Alquileres.
- Auxiliares.
- Margenes e impuestos.

Y tambien por oficio:

- Pintura.
- Electricidad.
- Fontaneria.
- Carpinteria.
- Suelos.

Esto ayuda a encontrar y editar precios sin convertirlo en catalogo infinito.

### 6. Crear trabajos tipo desde una prueba real

Una forma facil de explicar Cotiza:

1. El usuario crea un presupuesto una vez.
2. Ajusta lineas y cantidades.
3. Si le sirve, pulsa "Guardar como trabajo tipo".

Asi la plantilla nace de un caso real, no de una configuracion abstracta.

### 7. Usar nombres menos tecnicos en la interfaz

Posible cambio de lenguaje:

- "Precios" puede seguir igual.
- "Rendimientos" podria explicarse como "Consumos y tiempos".
- "Trabajos tipo" podria explicarse como "Plantillas de trabajos".
- "Calcular base" podria ser "Crear presupuesto base".
- "Multiplicador" deberia llevar ayuda visible: "extra o merma, por ejemplo 1,08 para sumar 8%".

No hace falta cambiar toda la arquitectura; alcanza con explicar mejor.

## Mejoras recomendadas para Cotiza segun este aprendizaje

### Para la demo clara

- Agregar una tarjeta de ayuda en "Precios": "Carga lo que compras o cobras por unidad".
- Agregar una tarjeta de ayuda en "Rendimientos": "Carga cuanto se consume o tarda por m2, unidad o metro lineal".
- Agregar una tarjeta de ayuda en "Trabajos tipo": "Combina precios y tiempos para no repetir presupuestos".
- Mostrar una explicacion del calculo despues de pulsar "Calcular base".
- Agregar ejemplos por oficio para que la prueba no dependa solo de pintura.

### Para despues de validar

- Asistente "Crear mi primer trabajo tipo".
- Boton "Guardar este presupuesto como plantilla".
- Importacion simple desde Excel/CSV.
- Entrada rapida por texto o voz para sugerir una plantilla, siempre editable.
- Comparador de coste interno vs precio cliente mas claro.

## Posicionamiento recomendado para Cotiza

Cotiza deberia explicarse asi:

> Cotiza ayuda a autonomos y pequenas empresas de oficios a crear presupuestos claros en minutos usando sus propios precios, rendimientos y trabajos tipo. No reemplaza el criterio profesional: ordena el calculo y deja el presupuesto editable.

## Diferenciacion principal

Cotiza no debe venderse como:

- ERP de construccion.
- IA que inventa presupuestos.
- Software tecnico de mediciones de obra.
- Facturador generico.

Cotiza debe venderse como:

- Calculadora profesional de presupuestos.
- Biblioteca controlada de precios, rendimientos y trabajos tipo.
- Herramienta simple para enviar presupuestos mas rapido.
- Producto visual, claro y editable para oficios.

## Hipotesis para validar en prueba privada

- El usuario entiende la diferencia entre precio, rendimiento y trabajo tipo.
- El usuario valora tener sus propios trabajos tipo.
- El usuario prefiere editar una base calculada antes que escribir todo desde cero.
- El usuario no necesita facturacion en la primera prueba.
- El usuario quiere imprimir o enviar un PDF limpio.
- El usuario usaria la app desde movil durante o despues de una visita.
- El usuario confia mas si ve de donde sale cada calculo.

## Acciones recomendadas

- Mantener la demo simple y centrada en presupuesto.
- Agregar mas datos ficticios realistas por oficio antes de mostrarla a mas personas.
- Preparar una tabla corta de comparacion para la web futura.
- En la prueba privada, preguntar que usa hoy: Excel, WhatsApp, Word, papel, app de facturacion o software de obra.
- Preguntar cuanto tarda hoy en enviar un presupuesto despues de ver un trabajo.
- Preguntar que parte le da mas miedo: olvidar partidas, calcular mal materiales, perder margen o presentar feo el presupuesto.
